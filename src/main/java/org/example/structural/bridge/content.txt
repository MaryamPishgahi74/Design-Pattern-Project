الگوی Bridge زمانی استفاده می‌شود که یک چیز دو بُعدِ توسعه‌پذیر دارد و نمی‌خواهیم این دو بُعد در هم قفل شوند.
این الگو Abstraction را از Implementation جدا می‌کند تا هر کدام مستقل توسعه داده شوند.
----------------------------
چهار نوع توسعه را از هم جدا کردیم

نوع شکل → Circle, Square

نوع رسم → Red, Blue, …
بدون اینکه مجبور باشیم برای هر ترکیب یک کلاس بسازیم.

اگر فردا بخواهی:

۱۰ نوع شکل اضافه کنی

۱۰ نوع رسم اضافه کنی

فقط 20 کلاس داری، نه 100!

هر بخش مستقل رشد می‌کند:

می‌توانیم شکل‌ها را تغییر دهیم بدون دست زدن به رنگ‌ها

یا رنگ‌ها را عوض کنیم بدون تغییر در شکل‌ها

 یک مثال واقعی و ملموس‌تر

فرض کن داری یک سیستم پرداخت می‌زنی:

Abstraction (نقش اصلی)

Payment → CreditCardPayment, BankPayment

Implementation (کانال انتقال)

PayPalGateway

StripeGateway

LocalBankGateway

این‌ها را با Bridge به هم وصل می‌کنی تا از انفجار کلاس جلوگیری شود.
